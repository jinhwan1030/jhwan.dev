## 왜 만들었나

아이를 키우다 보면 매일 아침 같은 고민을 하게 된다. 오늘 나가도 될까? 미세먼지는? 자외선은? 너무 덥진 않을까?

날씨 앱을 열고, 미세먼지 앱을 따로 열고, 자외선 지수를 검색하고... 이 과정이 매번 번거로웠다. 그래서 그냥 만들기로 했다. 장소를 검색하면 외출 적합도를 0~100점으로 보여주는 서비스.

## 기술 스택

```
FastAPI (Python)
Docker + Docker Compose
Nginx (프론트 static 서빙 + API 프록시)
라즈베리파이 4B (직접 서버 운영)
Cloudflare DNS + NPM SSL
```

데이터는 세 곳에서 가져온다.

- **기상청 단기예보 API** — 기온, 강수확률
- **한국환경공단 에어코리아 API** — 통합대기환경지수(CAI)
- **기상청 자외선지수 API** — UV Index
- **카카오 로컬 API** — 장소명 → 위경도 변환

## 점수 계산 로직

총 100점 만점. 각 항목별로 공신력 있는 기준을 적용했다.

| 항목 | 만점 | 기준 |
|------|------|------|
| 기온 | 30점 | 기상청 폭염·한파 특보 기준 |
| 강수확률 | 30점 | 기상청 단기예보 강수확률 |
| 자외선 | 25점 | WHO·질병관리청 UV Index 5단계 |
| 미세먼지 | 15점 | 환경부 통합대기환경지수(CAI) |

예를 들어 기온은 10~28°C면 만점(30점), 33~38°C면 9점, 38°C 초과면 0점이다.

## 힘들었던 점

### 1. 측정소 탐색 속도 문제

처음엔 에어코리아 API를 직접 호출해서 가장 가까운 측정소를 찾았다. TM 좌표 변환 → 근접 측정소 조회 순서인데, 이 과정에서 API 응답이 느려 카카오 오픈빌더 타임아웃(5초)에 걸렸다.

해결책은 에어코리아에서 전국 673개 측정소 목록(위경도 포함)을 미리 받아서 `stations.json`으로 로컬에 저장하고, 요청마다 직선거리 계산으로 가장 가까운 측정소를 찾는 방식으로 바꿨다. API 호출 한 번이 줄어들면서 응답 속도가 크게 개선됐다.

### 2. 카카오 오픈빌더 발화 매칭 한계

처음엔 카카오톡 챗봇으로 시작했다. 근데 오픈빌더는 사전에 등록된 발화 패턴과 일치해야 응답하는 구조라, "양양공항", "퇴계우성아파트" 같은 자유 지명 입력을 커버하기가 구조적으로 불가능했다.

결국 웹앱으로 방향을 전환했다. 오히려 더 나은 선택이었다. 사용자가 어떤 지명이든 입력할 수 있고, 카카오 API가 알아서 위경도로 변환해준다.

### 3. 공공데이터 API 첫 응답 지연

기상청과 에어코리아 API가 처음 호출 시 응답이 느린 경우가 있었다. 캐시가 없는 새로운 지역 첫 조회 시 3개 API를 `asyncio.gather`로 병렬 호출해도 10초 이상 걸리는 경우가 생겼다.

프론트에서 28초 타임아웃 + 자동 재시도(최대 2회) 로직을 넣어서 사용자 경험을 개선했다. 두 번째 요청부터는 캐시(TTL 60분)에서 즉시 응답한다.

## 서비스 구조

```
babyweather.jhwan.dev
    ↓
Cloudflare DNS
    ↓
라즈베리파이 4B (NPM 443)
    ↓
Docker Compose
    ├── nginx (프론트 static + /api 프록시)
    └── FastAPI (실제 API 처리)
```

프론트(HTML/CSS/JS)와 백엔드(FastAPI)를 같은 Docker Compose로 묶어서 운영한다. nginx가 `/api/` 경로는 FastAPI로 프록시하고, 나머지는 static 파일을 서빙한다. CORS 문제도 없고 구조도 단순하다.

## 결과

[babyweather.jhwan.dev](https://babyweather.jhwan.dev) 에서 직접 써볼 수 있다.

장소를 검색하면 외출 지수 점수와 함께 기온·강수·자외선·미세먼지 세부 점수, 그리고 각 점수의 기준 출처를 툴팁으로 확인할 수 있다. PWA로 만들어서 홈 화면에 앱처럼 설치도 된다.

아직 초단기예보 혼합, 연령별 맞춤 기준 등 고도화할 부분이 남아있다. 실제로 매일 쓰면서 계속 개선할 예정이다.
